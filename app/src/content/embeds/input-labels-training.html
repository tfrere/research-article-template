How it works:

input_ids: Complete sequence of input tokens (user + assistant)

labels: Target tokens for training

Masked labels (-100): User tokens are masked because the model should not learn to predict them

Active labels: Only assistant tokens are used to calculate training loss

Goal: The model learns to generate coherent assistant responses based on user context
