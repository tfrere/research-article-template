Comment ça fonctionne :

input_ids : Séquence complète des tokens d'entrée (utilisateur + assistant)

labels : Tokens cibles pour l'entraînement

Labels masqués (-100) : Les tokens utilisateur sont masqués car le modèle ne doit pas apprendre à les prédire

Labels actifs : Seuls les tokens de l'assistant sont utilisés pour calculer la perte d'entraînement

Objectif : Le modèle apprend à générer des réponses d'assistant cohérentes en se basant sur le contexte utilisateur
